Production extraction — CIV pilot, frozen schema v0.5, annexe fiscale. Governing docs fixed in project files: coverage map, processing plan, decision rules (incl. §7 scope, §8 program precedent, §9 lineage-deferral, §10 reconciliation-deferred, §11 cost-recovery, §11.1 collector-share, §12 nested keys), assumptions register, evidence dictionary v0.5, gold standard v2. Read and apply them. Do not reopen settled decisions. Production run: output is a deliverable, not a diagnostic.
── DOCUMENT PARAMETER (set per file) ──
document_id: [AF2025]; document_type: annexe_fiscale; document_year: [2025]; file: [attached].
── PREPROCESSING GATE (certify BEFORE extracting; if any check fails, STOP and report) ──
(1) Format: header bytes = %PDF (or state actual format — a mislabeled text/zip container is a known corpus hazard; note that the app-upload path can decompose a genuine searchable PDF into a zip-of-images, so certify the CANONICAL file's format, not the uploaded artifact's). (2) Encoding: text extracts as clean UTF-8 French, accents intact (é/à/è not mojibake) — the assignment-grammar sweep keys on accented terms, so mojibake causes silent false nulls. (3) Year: the document's internal title year matches document_year (wrong-year substitution is a known hazard). (4) Legibility: a mid-document sample page returns coherent French; corrupt font layer is a known hazard. Report the certified state in one line before proceeding.
── OPERATIVE TEST (binding, apply even if retrieval is imperfect) ──
An earmark requires BOTH (a) a source levy/tax/redevance/contribution/prélèvement/parafiscal charge AND (b) a verbatim clause assigning its proceeds — in whole or in part — to a specific named recipient. Extract-broad, filter-later: record EVERY assignment to a specific named recipient (fund, organism, ministry, tier of government, sector, programme, supranational body), marking is_purpose_restricted (1 = earmark subset; 0 = general-purpose recipient, e.g. a tier's general budget — still recorded). A fund merely existing is not sufficient. The ENACTING text governs; the exposé des motifs is motivation, not assignment — never infer an unnamed destination. §11: a split pricing the service the payer is consuming (administrative cost-recovery, processing-officer remuneration) is allocation_nature = cost_recovery_component, not an earmark. §11.1: a collecting body's retained share is cost_recovery_component only if it funds administering that specific levy, else proceeds_share. Only the pure general-State-budget residual uses beneficiary_type = general_budget.
── UNIT OF ANALYSIS (recurring pitfall) ──
The unit is the INSTRUMENT, not the article. French drafting bundles several unrelated taxes under one article number (e.g. one article creating a general-budget tax AND an earmarked tax). Decompose each article to its distinct instruments; classify is_purpose_restricted per instrument, never per article.
── STEP 1: ARTICLE ENUMERATION (completeness scaffold — do this FIRST, before any extraction) ──
Enumerate every article in the annexe by its number and title, as an ordered checklist covering the document start to end (annexes number articles sequentially; the count varies by year — do not assume a fixed number). This checklist is the completeness contract: every article must be accounted for in Step 3. Report the total article count and the first/last article numbers so coverage is auditable.
── STEP 2: FUND-AGNOSTIC RETRIEVAL (three-layer, discovery-preserving) ──
Layer 1 — assignment-grammar sweep across the WHOLE document for the act of assignment, regardless of destination name: affecté/affectation, au profit de, réparti/répartition, quote-part, quotité, produit de la taxe, est financé par le produit de, reversé, alimente, compte spécial/compte d'affectation, ristourne. Include source-side cues (produit de, recouvré, perçu) and destination-side cues (fonds, bénéficiaire). The fund is the OUTPUT of the search, never the input — never use a known-fund checklist (that would only confirm known earmarks and miss new ones). Layer 2 — heading pass: scan article/section titles for assignment language (répartition, affectation, fonds, a named account). Layer 3 — recall audit (mandatory deliverable, see below).
── STEP 3: EXTRACT ──
For each article, populate the evidence table for its earmark-bearing clauses. Source and allocation as distinct row types (independently-attested halves of the pipe — a source may have no stated destination, and a destination clause may sit apart from its source; record each as it appears). Natural keys on every row (document_id, legal_article, official_name verbatim, verbatim_excerpt, page_start/page_end); instrument_id/pair_id blank (assigned in reconciliation). Every row carries a verbatim excerpt containing the operative language.
Apply all v0.5 mechanics: rate_value scalar or null; rate_is_schedule + rate_schedule_detail for banded; rate_basis ∈ {stated, banded, deferred_arrete, cross_reference, not_applicable}; rate_type; share_value scalar or null; share_is_schedule + share_schedule_detail; share_type; share_basis ∈ {stated, whole_proceeds_implied, deferred_arrete}; share_level (1 = share of gross instrument; 2 = share of a named sub-pool; 3+ if deeper) and share_pool (verbatim name of the parent pool for share_level ≥ 2, null at level 1) — a self-contained descriptor, NOT a row-pointer, so extraction stays id-free; destination verbatim; beneficiary_type; allocation_nature ∈ {proceeds_share, cost_recovery_component}; is_purpose_restricted; assignment_type; partial_key (1 if only amended tiers restated, remainder elsewhere); tax_instrument; tax_base_detail; source_internal_inconsistency where the text is self-inconsistent; mechanism left to the derived instrument layer (not populated here). Lineage (§9): predecessor_ref and predecessor_relation ∈ {amends, replaces, merges, splits, none} ONLY from what the text states — capture the signal, make NO cross-document judgment; change_type and structural_break provisional; human_validation_status = unchecked.
── NESTED KEYS (§12) ──
When an article splits proceeds in two levels — a share goes to a pool, and that pool is itself re-split (e.g. 90% to a DGAM pool, then that 90% split 80/9/1/4/3/3) — record the top-level shares at share_level = 1 and the sub-shares at share_level = 2 with share_pool naming the parent pool verbatim. The 100%-sum check applies WITHIN a (level, pool), never across levels: a level-2 set sums to 100% of its pool, not of the gross instrument. Do not flatten the two levels into one; do not mix their shares in any sum.
── NO-INVENTION & NULL DISCIPLINE (from decision rules) ──
Destination not named in the enacting clause → destination null, ai_confidence = medium, never a plausible-but-unstated fund. Rate/share deferred or banded → scalar null + the appropriate _basis/_is_schedule flag, never a fabricated number and never text in a numeric field. general_budget residual rows recorded for the 100%-sum check but excluded from earmark analysis.
── OUTPUT ──
Write the evidence table to .xlsx (columns in v0.5 dictionary order; plain formatting, no decorative cell fills). Then report inline:
(1) COVERAGE REPORT — the Step-1 article checklist annotated: for EVERY article, either the row(s) it produced or "checked — no earmark clause." Every article accounted for; total = enumerated count. This is the proof the whole document was read end to end.
(2) RECALL AUDIT — list every assignment-grammar hit from the Layer-1 sweep and confirm each is either captured as a row or dismissed with a one-line reason (e.g. "expenditure credit, no source" / "membership dues paid out"). No hit left unaccounted.
(3) FIELD-EXERCISE NOTE — which v0.5 fields fired; flag every allocation_nature/§11.1 call, every cross_reference/banded rate, every is_purpose_restricted = 0 row, and every share_level = 2 (nested) row for my sign-off.
(4) SCHEMA-STRESS NOTE — only genuinely NEW gaps (a clause fitting no column, an unanticipated null, a judgment the rules don't cover). A real one means STOP — the schema needs revision before continuing the corpus.
(5) CONFIDENCE REASONS — per non-high-confidence row, the reason typed as: document problem (unnamed destination / deferred value / interleaving / nested-level ambiguity) vs. schema problem.
(6) GOLD SCORING — score against any gold-standard cases anchored in this document (recall + any invented field).
── DISCIPLINE ──
Do not summarize the annexe or editorialize. Extract what the text states, anchored and verbatim. When uncertain, record the uncertainty in notes — never resolve it silently. The coverage report and recall audit together are the guarantee that no earmark-bearing clause in any article, front or back of the document, was missed.
